A inteligência artificial transformou radicalmente o desenvolvimento de software, e o GitHub Copilot tem sido um dos protagonistas dessa revolução, atuando como um assistente de codificação inteligente para milhões de desenvolvedores. No entanto, o cenário para os usuários do Copilot mudou drasticamente a partir de 1º de junho de 2026, com a transição para um novo sistema de cobrança baseado em tokens. Essa alteração, anunciada pelo GitHub em 27 de abril de 2026, está gerando um forte descontentamento na comunidade, com muitos desenvolvedores relatando aumentos de custo que variam de 10 a 60 vezes em relação aos valores anteriores.

**Resposta Rápida (TL;DR):** O GitHub Copilot mudou para cobrança baseada em tokens a partir de 1º de junho de 2026, substituindo o modelo de "solicitações premium" por "Créditos de IA do GitHub". Essa transição, motivada pela crescente demanda computacional de recursos mais avançados ("agentic"), resultou em aumentos de custos projetados de 10 a 60 vezes para usuários intensivos, gerando um forte descontentamento na comunidade de desenvolvedores.

## A Virada para a Cobrança por Uso: O Fim das Unidades de Solicitação Premium

Até o final de maio de 2026, os planos do GitHub Copilot operavam com um sistema que incluía "unidades de solicitação premium" (PRUs) para interações mais avançadas, enquanto a maioria das funcionalidades era acessada por uma taxa mensal fixa. Esse modelo, segundo o GitHub, tornou-se insustentável diante da evolução da ferramenta. Em 1º de junho de 2026, todos os planos do GitHub Copilot migraram para um sistema de cobrança baseado em uso, onde as PRUs foram substituídas por "Créditos de IA do GitHub".

Agora, cada interação com o Copilot, com exceção das sugestões de código e "Next Edit", consome esses créditos, que são calculados com base no consumo de tokens – incluindo tokens de entrada, saída e até mesmo tokens armazenados em cache. Cada Crédito de IA do GitHub equivale a US$ 0,01.

### Por Que a Mudança? A Ascensão do Copilot "Agentic"

O GitHub justifica a alteração na precificação como um passo necessário para manter a sustentabilidade e a confiabilidade do serviço. De acordo com Mario Rodriguez, CPO do GitHub Copilot, a ferramenta não é a mesma de um ano atrás. Ela evoluiu de um assistente simples no editor para uma "plataforma agentic" capaz de realizar sessões de codificação complexas e multifacetadas, que operam sobre repositórios inteiros e utilizam modelos de IA de ponta.

Essa evolução para um uso "agentic" demanda significativamente mais poder de computação e inferência. Anteriormente, uma pergunta rápida no chat e uma sessão de codificação autônoma de várias horas poderiam custar o mesmo ao usuário, com o GitHub absorvendo grande parte dos custos crescentes de inferência. O modelo de cobrança baseado em uso visa alinhar o preço ao consumo real, permitindo ao GitHub cobrir os crescentes custos operacionais e manter a qualidade do serviço a longo prazo.

### Como Funciona o Novo Modelo: Tokens e Créditos de IA

Apesar da mudança no modelo de cobrança, os preços base das assinaturas mensais do GitHub Copilot, para a maioria dos planos, permanecem os mesmos: Copilot Pro a US$ 10/mês, Pro+ a US$ 39/mês, Business a US$ 19/usuário/mês e Enterprise a US$ 39/usuário/mês. No entanto, esses valores agora correspondem a uma alocação mensal de Créditos de IA.

Por exemplo, o plano Copilot Pro agora inclui US$ 10 em Créditos de IA (equivalente a 1.000 créditos), e o Pro+ inclui US$ 39 em Créditos de IA (3.900 créditos). Usuários de planos anuais, por sua vez, permanecerão no modelo de PRUs até a expiração de suas assinaturas, mas com multiplicadores de modelo aumentados. Após a expiração, eles serão transferidos para o plano Copilot Free, com a opção de fazer upgrade para um plano mensal pago. O GitHub também introduziu o plano Copilot Max, que custa US$ 100/mês e oferece 20.000 créditos (US$ 200 em uso) para usuários de alto volume, disponível como upgrade para assinantes existentes e com novas inscrições a serem abertas "nas próximas semanas".

É crucial destacar que as sugestões de código e as sugestões de "Next Edit" – as funcionalidades básicas de preenchimento automático que tornaram o Copilot popular – continuam ilimitadas e não consomem Créditos de IA em todos os planos pagos. Contudo, recursos como Copilot Chat, modo agente, Copilot CLI, Copilot Spaces e revisão de código do Copilot (que também consome minutos do GitHub Actions) agora dependem dos créditos disponíveis. Anteriormente, quando os usuários esgotavam as PRUs, o Copilot poderia recorrer a um modelo de menor custo para continuar funcionando. Sob o novo sistema, essa experiência de fallback não está mais disponível; o uso será interrompido se os créditos e o orçamento do administrador forem esgotados.

Para empresas e organizações, o GitHub introduziu o conceito de uso agrupado de créditos, permitindo que os créditos não utilizados por um usuário sejam compartilhados pela organização, otimizando o consumo. Além disso, administradores ganharam novos controles orçamentários para definir limites em nível de empresa, centro de custo e usuário. Clientes Copilot Business e Enterprise existentes também receberão créditos promocionais em junho, julho e agosto (US$ 30 para Business, US$ 70 para Enterprise).

## O Choque nos Custos: Aumentos Drásticos para Desenvolvedores

Desde que o novo modelo de cobrança entrou em vigor em 1º de junho de 2026, a comunidade de desenvolvedores tem expressado forte "fúria" e "indignação". Relatos em fóruns como o GitHub Community e o Reddit, e noticiados por veículos como o TechTimes e o Tecnoblog, indicam aumentos de custo "drásticos" e "desproporcionais".

Usuários intensivos estão enfrentando projeções de contas que saltam de forma exponencial. Por exemplo, um usuário do Copilot Pro+ que antes pagava US$ 39 por mês projetou que sua conta mensal subiria para US$ 847 com o mesmo padrão de uso, levando-o a cancelar o serviço. Outros relatos incluem 13% dos créditos de um plano sendo consumidos em menos de uma hora, ou 71% em apenas um dia. Em alguns casos extremos, desenvolvedores que pagavam entre US$ 25 e US$ 50 por mês estão vendo projeções de custos que chegam a US$ 3.000 para o mesmo nível de utilização.

Tanto o [TechTimes](https://www.techtimes.com/articles/307436/20260601/github-copilot-pricing-change-drives-backlash-agentic-bills-jump-10x-50x-power-users.htm) quanto o Business Insider confirmam que os relatórios da comunidade documentam aumentos de custos projetados de 10x a 50x para desenvolvedores que executam sessões de codificação "agentic". A razão para esses aumentos reside no consumo imprevisível de tokens. Modelos de IA mais avançados, como o GPT-5.5 (em comparação com o GPT-5.4 nano, por exemplo), podem custar ordens de magnitude mais por milhão de tokens de saída, transformando uma abordagem de "vamos ver o que acontece" em um esgotamento rápido dos créditos mensais.

### Impacto para Usuários Individuais e Corporativos

Para os usuários individuais, o impacto é imediato e palpável. Muitos sentem que o "paraíso" da precificação fixa, onde podiam usar o Copilot como um "Netflix" de código, acabou. A falta de previsibilidade nos custos é um ponto central da frustração, já que é difícil estimar quantos créditos uma sessão de codificação consumirá, especialmente com os recursos "agentic". Quando os créditos acabam, os recursos premium do Copilot são pausados, a menos que o usuário compre créditos adicionais ou faça upgrade para um plano superior.

No cenário corporativo, embora haja a vantagem do agrupamento de créditos e controles de orçamento, as empresas também precisarão adaptar suas estratégias. O GitHub enfatiza que esse é um passo em direção à "economia de computação em nuvem" para ferramentas de codificação de IA, sugerindo que as empresas devem aplicar as mesmas práticas de governança de custos que usam para a nuvem. Isso inclui monitoramento, alertas e otimização para gerenciar o consumo do Copilot de forma mais ativa do que antes.

### A Questão da Transparência e Previsibilidade

A mudança de um modelo de "assinatura fixa" para um de "uso por token" levanta questões sobre a transparência e a previsibilidade. Muitos desenvolvedores expressam a sensação de que, embora o GitHub justifique a mudança pela sustentabilidade, a forma como os custos escalam é excessiva e difícil de prever. A complexidade da precificação por modelo, por tipo de token (entrada, saída, cache) e por tipo de interação (chat, agente, revisão de código) torna o cálculo do custo final uma tarefa árdua para o usuário comum.

Isso pode levar os desenvolvedores a mudarem seus hábitos de uso, talvez limitando as sessões "agentic" ou optando por modelos menos custosos, ou até mesmo buscando alternativas. Alguns já estão explorando outras IAs de codificação que podem oferecer modelos de precificação mais favoráveis, como o DeepSeek, que em um exemplo foi estimado em cerca de 7 centavos para 15 milhões de tokens, ilustrando a grande disparidade de preços que pode existir entre os provedores.

## Além da Cobrança: O Futuro da Experiência Copilot

Apesar do alvoroço em torno da precificação, o GitHub mantém que o Copilot continua sendo a melhor ferramenta para "codificação agentic". A empresa também ressalta que a flexibilidade de modelos (OpenAI, Anthropic, Google, xAI) dentro de uma única assinatura, sem a necessidade de gerenciar várias chaves de API ou relações de faturamento, ainda oferece valor significativo, especialmente para grandes empresas.

Os novos controles de orçamento para administradores e o agrupamento de créditos são funcionalidades importantes para tentar mitigar o impacto nas organizações, permitindo um gerenciamento mais eficiente dos recursos e evitando que usuários menos ativos deixem créditos "parados".

No entanto, a era do acesso ilimitado "all-you-can-eat" para a maioria dos recursos avançados do Copilot parece ter chegado ao fim. Desenvolvedores agora precisam ser mais intencionais sobre como e com que frequência usam os recursos do Copilot que consomem créditos. Essa mudança reflete uma tendência mais ampla no mercado de IA, onde o custo computacional real da execução de modelos avançados está sendo repassado aos usuários, marcando uma transição dos modelos de negócios de software como serviço (SaaS) para os de computação em nuvem (CaaS).

A comunidade agora aguarda para ver como essa mudança se estabilizará. Será que os desenvolvedores se adaptarão, ou a frustração levará a uma migração em massa para outras ferramentas e modelos de IA? A previsibilidade e a comunicação transparente dos custos serão cruciais para que o GitHub Copilot mantenha sua relevância e valor no longo prazo.

## Perguntas Frequentes (FAQ)

### O que mudou na cobrança do GitHub Copilot?
A partir de 1º de junho de 2026, o GitHub Copilot passou a ser cobrado com base no consumo de "Créditos de IA do GitHub", que são gastos conforme o uso de tokens (entrada, saída, cache) e o modelo de IA utilizado, substituindo o antigo sistema de "unidades de solicitação premium".

### Quais funcionalidades consomem Créditos de IA?
Funcionalidades como Copilot Chat, modo agente, Copilot CLI, Copilot Spaces, Spark e revisão de código do Copilot consomem Créditos de IA. Sugestões de código e "Next Edit" permanecem ilimitadas e gratuitas para todos os planos pagos.

### Os preços das assinaturas aumentaram?
Os preços base dos planos (Pro, Pro+, Business, Enterprise) não foram alterados. No entanto, esses valores agora representam uma alocação mensal de Créditos de IA, e usuários intensivos podem incorrer em custos adicionais significativamente mais altos caso ultrapassem essa alocação.
